# e2e/docs/

Deep dives that don't fit in `e2e/README.md`: hermetic-runner internals
(`scripts/e2e.sh` mechanics, alt ports), failure-artifact triage,
agent-browser CLI cookbook, locator strategy notes.

`e2e/README.md` stays the entry point (how to run, coverage table). Promote
a topic here once it grows past ~30 lines or wants its own diagram. Link new
files from `e2e/AGENTS.md` via the `Read … when …` block.

Note: `e2e/specs/` is the spec folder (already populated with
`NN-name.flow.json`); it doesn't need its own README placeholder.
